## Why Forma?

Most form libraries manage state imperatively — you write code to control every behavior. Forma takes a different approach: **forms are data, not code.**

A Forma spec is a portable JSON document with [FEEL (Friendly Enough Expression Language)](https://docs.camunda.io/docs/components/modeler/feel/what-is-feel/) expressions for conditional logic. This means your form definitions can be stored in a database, generated by AI, built with a visual editor, or shared across teams — independent of any UI framework.

- **Spec-driven** — Define structure, validation, visibility, and computed fields in a single JSON document
- **FEEL expressions** — Human-readable conditional logic based on an [actual standard](https://www.omg.org/spec/DMN/), not a custom DSL
- **Headless** — Bring your own UI components, full control over rendering
- **Framework-agnostic core** — React integration today, anything tomorrow

## Quick Start

### Installation

```bash
# Core runtime (required)
npm install @fogpipe/forma-core

# React integration (optional)
npm install @fogpipe/forma-react
```

### Define a Form

```typescript
import type { Forma } from "@fogpipe/forma-core";

const registrationForm: Forma = {
  version: "1.0",
  meta: {
    id: "registration",
    title: "User Registration",
    description: "Create a new account",
  },
  schema: {
    type: "object",
    properties: {
      email: { type: "string", format: "email" },
      age: { type: "integer", minimum: 18, maximum: 120 },
      hasLicense: { type: "boolean" },
      licenseNumber: { type: "string" },
    },
    required: ["email", "age"],
  },
  fields: {
    email: { type: "email", label: "Email Address" },
    age: { type: "integer", label: "Age" },
    hasLicense: {
      type: "boolean",
      label: "Do you have a driver's license?",
      // Only show if user is 16 or older
      visibleWhen: "age >= 16",
    },
    licenseNumber: {
      type: "text",
      label: "License Number",
      // Only visible and required if user has a license
      visibleWhen: "hasLicense = true",
      requiredWhen: "hasLicense = true",
    },
  },
  fieldOrder: ["email", "age", "hasLicense", "licenseNumber"],
  computed: {
    isAdult: { expression: "age >= 18", label: "Adult Status" },
  },
};
```

### Render with React

The quickest path is `DefaultFormRenderer` — pre-built, styled, accessible components out of the box:

```tsx
import { DefaultFormRenderer } from "@fogpipe/forma-react/defaults";
import "@fogpipe/forma-react/defaults/styles.css";

function App() {
  return (
    <DefaultFormRenderer
      spec={registrationForm}
      onSubmit={(data) => console.log("Submitted:", data)}
    />
  );
}
```

Need full control? Use `FormRenderer` with your own components:

```tsx
import { FormRenderer } from "@fogpipe/forma-react";
import { myComponents } from "./components";

<FormRenderer
  spec={registrationForm}
  components={myComponents}
  onSubmit={(data) => console.log("Submitted:", data)}
/>;
```

### Use the Core Runtime Directly

```typescript
import {
  getVisibility,
  getRequired,
  validate,
  calculate,
} from "@fogpipe/forma-core";

const formData = {
  email: "user@example.com",
  age: 25,
  hasLicense: true,
  licenseNumber: "ABC123",
};

// Check which fields are visible
const visibility = getVisibility(formData, registrationForm);
// { email: true, age: true, hasLicense: true, licenseNumber: true }

// Check which fields are required
const required = getRequired(formData, registrationForm);
// { email: true, age: true, hasLicense: false, licenseNumber: true }

// Calculate computed values
const computed = calculate(formData, registrationForm);
// { isAdult: true }

// Validate the form
const result = validate(formData, registrationForm);
// { valid: true, errors: [] }
```

## FEEL Expression Examples

Forma uses FEEL expressions for dynamic behavior:

```typescript
// Visibility — show field conditionally
{ visibleWhen: "age >= 18" }
{ visibleWhen: 'country = "US"' }
{ visibleWhen: "hasInsurance = true and insuranceType = \"health\"" }

// Required — make field conditionally required
{ requiredWhen: "hasInsurance = true" }
{ requiredWhen: "orderTotal > 100" }

// Computed fields — calculate derived values
{ expression: "quantity * unitPrice" }
{ expression: 'if income > 50000 then "premium" else "standard"' }

// Validation rules
{
  validations: [
    { rule: "value >= 0", message: "Must be positive" },
    { rule: "value <= 100", message: "Cannot exceed 100" },
  ]
}
```
